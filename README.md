# Security Awareness: Keylogger Demonstration Project

## Overview

This repository contains a sample program intended for security-awareness and educational discussions about keylogging threats.

A keylogger is software that records keyboard input. Attackers may use keyloggers to capture:

* Usernames and passwords
* Personal messages
* Financial information
* Sensitive business data

Understanding how keyloggers operate helps defenders recognize, detect, and prevent such threats.

## Educational Objectives

This project can be used to discuss:

* How keyloggers collect input data
* Common data-exfiltration techniques
* Social-engineering risks
* Malware detection strategies
* Endpoint security best practices
* Ethical and legal considerations

## Security Concepts Demonstrated

At a high level, keyloggers often include:

1. Input Monitoring

   * Observing keyboard events generated by the operating system.

2. Data Buffering

   * Temporarily storing captured information.

3. Background Execution

   * Running continuously without interrupting normal user activity.

4. Data Transmission

   * Sending collected information to an external destination.

5. Platform-Specific Components

   * Using different methods depending on the operating system.

## Detection Indicators

Security teams may look for:

* Unexpected keyboard-hook activity
* Unknown background processes
* Unusual outbound network connections
* Suspicious persistence mechanisms
* Antivirus or EDR alerts
* Unauthorized access to accessibility features

## Defensive Measures

* Use reputable endpoint protection software.
* Keep operating systems updated.
* Enable multi-factor authentication (MFA).
* Review installed applications regularly.
* Restrict administrative privileges.
* Monitor outbound network traffic.
* Educate users about phishing and social engineering.

## Ethical and Legal Notice

Keyloggers should only be used in environments where all participants have provided explicit authorization and where usage complies with applicable laws, regulations, and organizational policies.

Unauthorized monitoring of users or systems may be illegal and unethical.

## High-level analysis of Keylogger.py

This script is designed to:

1.Collect keyboard input
    On Windows/Linux desktops it uses the pynput library to monitor key presses.
    On Android (Termux) it attempts several methods to gather input-related events.
2.Store captured data
    Keystrokes are appended to a global buffer (key_buffer).
    A thread-safe lock (buffer_lock) is used to prevent concurrent access problems.
3.Periodically transmit data
    A background thread wakes up every SEND_INTERVAL seconds.
    Buffered keystrokes are packaged into a report and transmitted.
4.Exfiltrate data through Telegram
    The script builds a Telegram Bot API request.
    Captured information is sent to a configured chat ID.
5.Fallback logging
    If transmission fails, information is written to a local file (.key_buffer.txt).
6.Support multiple platforms
    Windows/Desktop Linux: keyboard listener.
    Android/Termux: accessibility, input-event, and clipboard-monitoring approaches.
